# 双全景联合布局 MVP

## 目标

输入相邻区域 A、B 的两张全景图片。两个区域不限定必须共享“门”这一类别，只要存在同一个可通行开口、洞口、门洞或开放接口，并且 Bi-Layout 的 extended 分支能够通过该开口预测到对侧空间，就可以把这个开口作为联合布局的配准锚点。

当前实现分为两步：

1. 使用现有 `inference.py` 分别预测 A、B 的单房间布局和全景边界线。
2. 使用 `tools/join_room_layouts.py` 指定共享开口在两个房间墙面上的位置，输出联合布局和联合边界线。

新增的跨场景估计 MVP 提供了一个自动候选生成入口：

```text
tools/estimate_cross_scene_layout.py
```

它不依赖门类别检测输出，而是枚举两个房间的墙段配对，把每一对墙的中心片段视作可能的共享开口/接口，估计 B 到 A 的相对位姿，并按墙长一致性、尺度变化、房间重叠率和中心点侧向关系给候选排序。

## 需求边界

这里的“共享开口”是几何概念，不是语义类别。它可以是门、门洞、开放通道、宽开口、拱门或没有门扇的连通口。判断标准不是“是否被标注为 door”，而是：

1. A、B 两个全景来自可连通的相邻空间。
2. 两个单图预测中存在可以互相对应的墙面片段或开口片段。
3. extended 分支在该方向上表现出“可通过/向外延伸”的趋势，而不是被 enclosed 边界完全封死。
4. 融合后两个空间应位于共享开口两侧，且不产生明显大面积重叠或反向拼接。

因此，后续自动化应该优先利用 enclosed / extended 的差异来发现可通行接口：enclosed 提供当前房间的稳定边界，extended 提供穿过开口后的外延线索，两者共同决定是否进入跨场景融合估计。

## 当前限制

仓库当前没有可用的开口检测输出。`dataset/communal/read.py` 能读取门标注，但模型输出中没有 `doors` 或 `openings` 字段，`loss/object_loss.py` 仍是占位实现。

因此，第一阶段需要人工指定共享开口。后续加入开口检测器或 extended 可通行性检测器时，可以直接生成相同参数并复用联合布局模块。

当前新增的 `estimate_cross_scene_layout.py` 属于几何候选估计版。它不能保证找到真实开口，但可以给出 top-k 可能相邻的墙对。现在工具已经接入规则版 extended passability：如果输入 JSON 中包含 `biLayoutOutputs.depth` 和 `biLayoutOutputs.new_depth`，或额外传入 extended layout JSON，就会优先用 `new_depth - depth` 的外延差异生成共享开口候选；如果没有这些信息，则自动退回墙中心枚举。

## 运行方式

先对两张图片执行现有推理：

```bash
python inference.py \
  --img_glob "src/input/pair/*.jpg" \
  --cfg src/config/zind_simple.yaml \
  --output_dir src/output/pair \
  --post_processing manhattan
```

然后连接两个房间：

```bash
python tools/join_room_layouts.py \
  --layout_a src/output/pair/a_pred.json \
  --layout_b src/output/pair/b_pred.json \
  --door_a 1:0.25:0.55 \
  --door_b 3:0.30:0.60 \
  --output_dir src/output/pair_joint \
  --name a_b
```

开口参数格式为：

```text
墙编号:开口起点比例:开口终点比例
```

例如 `1:0.25:0.55` 表示共享开口位于第 1 面墙，从墙长的 25% 延伸到 55%。

## 输出

命令会生成：

```text
src/output/pair_joint/a_b.json
src/output/pair_joint/a_b_boundaries.svg
src/output/pair_joint/a_b_boundaries.png
```

JSON 包含：

- A、B 在同一世界坐标系中的房间边界点。
- A、B 的 `boundaryLines` 边界线段。
- `sharedOpening` 共享开口线段。
- `sharedDoor` 兼容旧版本字段，语义上同样表示共享开口。
- `roomBToWorld` 配准矩阵。
- 使用共享开口宽度估计的 B 房间尺度。

SVG 联合边界图始终输出，并在墙面中间显示 `A:墙编号` 和 `B:墙编号`。安装 OpenCV 的项目推理环境会额外输出 PNG。

默认情况下，工具使用同一个共享开口的宽度校准 B 房间尺度。如需保留两个单图预测原始尺度，添加 `--preserve_scale`。

## 自动估计跨场景候选

如果还没有开口检测结果，可以先运行几何候选估计：

```bash
python tools/estimate_cross_scene_layout.py \
  --layout_a src/output/pair/a_pred.json \
  --layout_b src/output/pair/b_pred.json \
  --output_dir src/output/pair_cross_scene \
  --name a_b \
  --top_k 8 \
  --anchor_ratio 0.30 \
  --simplify_tolerance 0.05 \
  --opening_threshold 0.25 \
  --passability_weight 1.0
```

如果 enclosed 与 extended 分支分别保存成两个 JSON，可以显式传入 extended 结果：

```bash
python tools/estimate_cross_scene_layout.py \
  --layout_a src/output/pair/a_enclosed_pred.json \
  --layout_b src/output/pair/b_enclosed_pred.json \
  --layout_a_extended src/output/pair/a_extended_pred.json \
  --layout_b_extended src/output/pair/b_extended_pred.json \
  --output_dir src/output/pair_cross_scene \
  --name a_b
```

输出：

```text
src/output/pair_cross_scene/a_b_candidates.json
src/output/pair_cross_scene/a_b_best_joint.json
src/output/pair_cross_scene/a_b_best_joint.svg
src/output/pair_cross_scene/a_b_best_joint.png
```

`a_b_candidates.json` 中每个候选包含：

- `wallA` / `wallB`: 估计的相邻墙编号。
- `openingA` / `openingB`: 用于对齐的墙内中心片段，格式与 `join_room_layouts.py` 的开口参数一致。
- `doorA` / `doorB`: 兼容旧版本字段，语义上同样表示开口片段。
- `roomBToWorld`: B 房间到 A 世界坐标系的相似变换矩阵。
- `score` / `confidence`: 几何候选分数和置信度。
- `metrics`: 墙长一致性、尺度惩罚、面积比例、重叠率等诊断指标。
- `openingEvidenceA` / `openingEvidenceB`: 开口候选来源、token 区间、passability 置信度和外延差异统计。

`passability.summary` 会说明本次是否使用了 extended 信息：

```text
hasExtendedDepth = true:
  使用 depth / new_depth 或 extended JSON 生成开口候选。

hasExtendedDepth = false:
  输入没有 extended 信息，自动退回墙中心枚举。
```

如果某个 layout JSON 是 256 个边界采样点而不是少量墙角点，工具会在墙段搜索前自动做 polygon simplification。候选 JSON 的 `layoutSimplification` 字段会记录原始墙段数、简化后墙段数和简化阈值。

这个工具的作用是把跨场景估计拆成一个可跑通的第一阶段：

```text
单图 layout JSON -> 候选共享开口/接口 -> 相对位姿 -> 联合布局可视化
```

## 后续自动化方向

完整自动化后续还需要新增可学习的开口/连通性检测分支：

1. 从训练数据提取开口中心、宽度和高度监督信号；如果没有显式开口标注，则从 extended / enclosed 差异中构造弱监督。
2. 当前已实现规则版 `new_depth - depth` passability，后续可以在模型中增加开口热力图、开口宽度回归头或可学习 passability score。
3. 在 A、B 候选开口之间加入 cross attention 共有区域匹配。
4. 将匹配结果转换成当前工具使用的墙编号和墙内比例。
