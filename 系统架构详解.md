## 二、数据张量维度变化追踪

### 2.1 训练一个Batch的完整流程

下面按照论文里常见的“张量形态 + 物理含义”方式写成 4 个阶段。

#### Stage 1. 物理输入空间

```
X_in ∈ R^{B×3×512×1024}
```

- `B`: batch size，对应一批全景图样本。
- `3`: RGB 三通道。
- `512×1024`: ERP 全景图分辨率，水平 1024 覆盖 $360^\circ$，垂直 512 表示整幅球面投影的离散采样。
- 这是最原始的图像张量，还没有任何语义压缩。

#### Stage 2. 视觉特征提取与高度坍缩

```
F_vis = HorizonNetFeatureExtractor(X_in)
F_vis ∈ R^{B×1024×256}
```

- 宽度从 `1024 → 256`，对应水平方向的离散经度序列被压缩为 256 个特征位置。
- 高度从 `512 → 64` 的结构信息被卷积和池化吸收，垂直几何不再以显式空间维存在。
- `1024` 维通道不是颜色通道，而是更高层的布局语义特征，包含墙、角点、开口、边界连续性等信息。
- 这一阶段可以理解为把 2D 图像“压成”一条沿水平方向排列的特征栅栏。

#### Stage 3. 序列化与 Transformer 上下文建模

```
F_seq = permute(F_vis)
F_seq ∈ R^{B×256×1024}
```

- 通过 `permute(0, 2, 1)`，张量从 `B×C×L` 变成 `B×L×C`。
- 这里 `L=256` 可视为序列长度，`C=1024` 可视为 embedding 维度。
- 这一步的目标是把全景图重写成序列形式，从而接入 Transformer 的全局注意力机制。
- 语义上，相当于把 256 个经度位置当作 256 个 token，每个 token 携带 1024 维特征。

位置编码与查询向量在这一阶段一起注入：

```
Pos ∈ R^{B×256×1024},  Query ∈ R^{256×512}
```

- `Pos` 提供“这个 token 来自哪里”的位置信息。
- `Query` 提供“模型应该关注什么”的可学习先验。
- 对应论文表述时，通常写成 `F_seq + Pos` 与 `Query` 共同送入 Transformer。

Transformer 解码后保持序列形态不变：

```
F_tf ∈ R^{B×256×1024}
```

- 这里维度不变，但每个位置已经获得全局上下文。
- 这一步是全景布局任务的核心，因为房间边界需要跨位置一致性约束。

#### Stage 4. 标量压缩与全局回归头

```
H_ratio = Linear(F_tf)
H_ratio ∈ R^{B×256×1}
```

- 这一阶段对应你写的“高度压缩”中间态。
- 需要特别说明的是：这里的 `256×1` 不是最终输出，而是“每个位置先压成一个标量响应”的中间表示。
- 语义上可以理解为：每个经度位置先给出一个高度相关的激活强度，然后再在序列维度上做全局汇聚。

随后继续压缩得到最终标量：

```
ratio ∈ R^{B×1}
```

- 这才是最终的房间高度比例预测。
- 如果你的论文笔记想保留“[B, 256, 1]”这一段，建议把它标成“中间标量响应”，避免和最终 ratio 混淆。

同理，深度分支的最终输出为：

```
depth ∈ R^{B×256}
new_depth ∈ R^{B×256}
```

- `depth` 表示原始布局分支的 256 点深度序列。
- `new_depth` 表示遮挡感知分支的 256 点深度序列。

### 2.2 训练一个Batch的完整张量流

```
X_in ∈ R^{B×3×512×1024}
   → F_vis ∈ R^{B×1024×256}
   → F_seq ∈ R^{B×256×1024}
   → F_tf  ∈ R^{B×256×1024}
   → H_ratio ∈ R^{B×256×1}
   → ratio ∈ R^{B×1}
   → depth / new_depth ∈ R^{B×256}
```

如果需要更贴近论文写法，可以直接写成：

$$
X \in \mathbb{R}^{B\times3\times512\times1024}
\xrightarrow{\text{feature extractor}}
F \in \mathbb{R}^{B\times1024\times256}
\xrightarrow{\text{permute}}
F' \in \mathbb{R}^{B\times256\times1024}
\xrightarrow{\text{Transformer}}
Z \in \mathbb{R}^{B\times256\times1024}
\xrightarrow{\text{regression head}}
\hat{y} \in \mathbb{R}^{B\times256\times1}
\xrightarrow{\text{global pooling}}
r \in \mathbb{R}^{B\times1}
$$

其中，`depth / new_depth` 是逐位置回归，`ratio` 是全局标量回归。

### 2.2 损失计算维度变化

```
预测 dt:
  depth:        (B=8, 256)
  ratio:        (B=8, 1)
  new_depth:    (B=8, 256)

地真 gt:
  depth:        (B=8, 256)
  ratio:        (B=8, 1)
  new_depth:    (B=8, 256)

L1Loss (depth) ─────────────────────────────────────
  |dt['depth'] - gt['depth']|
  ▼
  loss_depth:   (B=8, 256)
  ▼
  mean()
  ▼
  loss_depth:   scalar

L1Loss (ratio) ─────────────────────────────────────
  |dt['ratio'] - gt['ratio']|
  ▼
  loss_ratio:   (B=8, 1)
  ▼
  mean()
  ▼
  loss_ratio:   scalar

GradLoss (梯度) ────────────────────────────────────
  depth → 计算梯度 (dx, dy)
  ▼
  grad_loss:    scalar

加权求和 ───────────────────────────────────────────
  loss_total = w_depth * loss_depth
             + w_ratio * loss_ratio
             + w_grad * grad_loss
             + ...
  ▼
  loss_total:   scalar (用于反向传播)
```

### 3.2 ZindDataset 特殊处理

```
ZindDataset.__getitem__() 额外处理:
```

## 四、关键计算过程

### 4.1 深度值到XYZ转换

```
输入: depth向量 (256,)  ← 每个值代表该方向的深度(0-1)
      ratio: float     ← 房间高度

转换步骤:

Step 1: 构造UV坐标序列
  u = linspace(0, 1, 256)     # 水平均匀分布
  v = 0.5 (地平线)             # 固定在地平线
  uv = [(u_i, 0.5) for u_i in u]
  ▼ output: (256, 2)

Step 2: UV → 3D方向向量
  对每个uv坐标:
    lon = u * 2π     (经度 longitude)
    lat = (0.5 - v) * π  (纬度 latitude, 地平线处为0)
    
    x = sin(lon) * cos(lat)
    z = cos(lon) * cos(lat)
    y = sin(lat)
  ▼ output: directions (256, 3) 单位向量

Step 3: 方向 × 深度 = 3D点
  xyz = depth_values[i] * direction[i]
  
  相机高度标准化为1:
    xyz[:, 1] += 1.0  (加上相机高度)
  ▼ output: xyz (256, 3)

Step 4: 应用房间高度缩放
  xyz[:, 1] *= ratio  (乘以高度比例)
  ▼ output: 最终XYZ坐标 (256, 3)

示例:
  depth = [0.2, 0.25, 0.22, ...]
  ratio = 1.75
  ▼
  xyz = [
    [0.195, 1.35, 0.039],    # 第一个深度样本的3D点
    [0.244, 1.31, 0.051],
    [0.215, 1.33, 0.043],
    ...
  ]
```

### 4.2 corners转换为depth向量

```
输入: corners = [(u1, v1), (u2, v2), ..., (un, vn)]  (N个角点UV坐标)

转换步骤:

Step 1: 提取可见角点 (v >= 0.5)
  visible_corners = [(u, v) for (u, v) in corners if v >= 0.5]

Step 2: 从地面投影计算
  地面多边形 = 从相机(0.5, 0.5)看，所有可见角点形成的边界
  ▼ output: 多边形顶点序列

Step 3: 采样深度值
  对于256个均匀的u位置 u_i in linspace(0, 1, 256):
    在该u方向上，找到多边形的地面交点
    depth_i = 该交点到相机的距离（0-1标准化）
  ▼ output: depth (256,)

示例:
  corners = [[0.2, 0.6], [0.4, 0.8], [0.6, 0.5], [0.3, 0.4]]
  ▼ (只保留v>=0.5的)
  visible_corners = [[0.2, 0.6], [0.4, 0.8], [0.6, 0.5]]
  ▼ (计算深度采样)
  depth = [0.19, 0.21, 0.25, 0.24, ..., 0.18]  (256个值)
```

### 4.3 Manhattan布局拟合

```
输入: floor_xz = [(x1, z1), (x2, z2), ..., (xn, zn)]  (2D平面图点集)

拟合步骤:

Step 1: 边界检测
  检测所有点的凸包或轮廓

Step 2: 线段抽取
  识别主要水平(x方向)和竖直(z方向)的线段

Step 3: 直角检测和调整
  如果两条线段接近90°，强制其为90°
  目标: 所有线段都是轴对齐的

Step 4: 角点合并
  移除过于接近的角点
  消除短边和噪声

Step 5: 闭合检查
  确保多边形闭合
  调整最后一个点以连接到第一个点

输出: processed_xz = [(x1', z1'), ..., (xm', zm')]  (规则化后的角点)
      通常 m < n (点数减少)
```

## 五、关键模块的输入/输出详细表

### 5.1 各数据处理函数签名

```python
# 图像处理
read_image(path: str, shape=[512, 1024]) 
  → np.ndarray(512, 1024, 3)

# 标签处理
read_label(path: str)
  → Dict{corners: (N,2), ratio: float, ...}

read_new_label(path: str)
  → Dict{new_corners: (M,2), ...}

# 坐标转换
uv2xyz(uv: (N, 2), plan_y=0) 
  → (N, 3) 3D坐标

depth2xyz(depth: (256,), plan_y=1)
  → (256, 3) 3D坐标序列

xyz2depth(xyz: (N, 3), plan_y=1)
  → (256,) 深度值

corners2boundaries(ratio: float, corners_xyz: (N, 3), ...)
  → (L,), (L,) 地面和天花板边界UV

# 数据增强
PanoDataAugmentation.execute_aug(
    corners: (N, 2), 
    image: (H, W, 3), 
    new_corners: (M, 2) = None
)
  → corners', image', new_corners'

# 深度生成
BaseDataset.get_depth(corners, patch_num=256)
  → (256,) 深度向量

visibility_corners(corners: (N, 2))
  → (K, 2) 其中K <= N，v >= 0.5的角点

# 后处理
post_process(b_depth: (B, 256), type_name='manhattan')
  → (B, Ni, 3) 每个样本可能有不同的角点数

fit_layout(floor_xz: (N, 2), need_cube=False)
  → (M, 2) M为规则化后的角点数
```

## 六、重要配置参数对性能的影响

### 6.1 模型参数

| 参数 | 默认值 | 影响 |
|------|--------|------|
| `patch_num` | 256 | 深度采样精度。↑值→精度↑，计算量↑ |
| `feature_channel` | 1024 | 特征维度。↑值→容量↑，显存↑ |
| `embedding_channel` | 512 | 查询嵌入维度。影响Transformer内部维度 |
| `depth` (层数) | 6 | Transformer深度。↑值→建模能力↑，推理慢↑ |
| `win_size` | 8 | 滑动窗口大小。影响receptive field和计算量 |
| `height_compression_scale` | 8 | 高度压缩因子。512→256(压8倍) |

### 6.2 训练参数

| 参数 | 默认值 | 影响 |
|------|--------|------|
| `batch_size` | 16 | 批大小。↑值→梯度稳定性↑，显存需求↑ |
| `lr` | 1e-4 | 学习率。影响收敛速度和稳定性 |
| `epochs` | 100 | 训练轮数。更多轮→更好拟合，过拟合风险↑ |
| `weight_decay` | 1e-4 | L2正则。防止过拟合 |

### 6.3 数据参数

| 参数 | 默认值 | 影响 |
|------|--------|------|
| `augment` | True | 数据增强。提高泛化能力 |
| `shape` | [512, 1024] | 输入大小。影响特征提取 |

### 7.2 常见问题

```
问题1: depth值不收敛
  原因: 深度采样点过多(256)可能信号过稀疏
  解决: ↓patch_num, ↑学习率

问题2: 显存溢出
  原因: batch_size过大或模型太深
  解决: ↓batch_size, ↓depth, 启用gradient_checkpointing

问题3: 验证IoU突然下降
  原因: 过拟合或学习率过大
  解决: ↓学习率, ↑weight_decay, 数据增强

问题4: new_depth分支不学习
  原因: 两分支共享特征，loss权重不当
  解决: 调整criterion权重，检查梯度流
```
