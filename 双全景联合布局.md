# 双全景联合布局 MVP

## 目标

输入相邻房间 A、B 的两张全景图片。两个房间共用同一扇门，门是联合布局的配准锚点。

当前实现分为两步：

1. 使用现有 `inference.py` 分别预测 A、B 的单房间布局和全景边界线。
2. 使用 `tools/join_room_layouts.py` 指定共享门在两个房间墙面上的位置，输出联合布局和联合边界线。

## 当前限制

仓库当前没有可用的门检测输出。`dataset/communal/read.py` 能读取门标注，但模型输出中没有 `doors` 字段，`loss/object_loss.py` 仍是占位实现。

因此，第一阶段需要人工指定共享门。后续加入门检测器时，可以直接生成相同参数并复用联合布局模块。

## 运行方式

先对两张图片执行现有推理：

```bash
python inference.py \
  --img_glob "src/input/pair/*.jpg" \
  --cfg src/config/zind_simple.yaml \
  --output_dir src/output/pair \
  --post_processing manhattan
```

然后连接两个房间：

```bash
python tools/join_room_layouts.py \
  --layout_a src/output/pair/a_pred.json \
  --layout_b src/output/pair/b_pred.json \
  --door_a 1:0.25:0.55 \
  --door_b 3:0.30:0.60 \
  --output_dir src/output/pair_joint \
  --name a_b
```

门参数格式为：

```text
墙编号:门起点比例:门终点比例
```

例如 `1:0.25:0.55` 表示门位于第 1 面墙，从墙长的 25% 延伸到 55%。

## 输出

命令会生成：

```text
src/output/pair_joint/a_b.json
src/output/pair_joint/a_b_boundaries.svg
src/output/pair_joint/a_b_boundaries.png
```

JSON 包含：

- A、B 在同一世界坐标系中的房间边界点。
- A、B 的 `boundaryLines` 边界线段。
- `sharedDoor` 共享门线段。
- `roomBToWorld` 配准矩阵。
- 使用共享门宽度估计的 B 房间尺度。

SVG 联合边界图始终输出，并在墙面中间显示 `A:墙编号` 和 `B:墙编号`。安装 OpenCV 的项目推理环境会额外输出 PNG。

默认情况下，工具使用同一扇门的宽度校准 B 房间尺度。如需保留两个单图预测原始尺度，添加 `--preserve_scale`。

## 后续自动化方向

完整自动化需要新增门检测分支：

1. 从训练数据提取门中心、宽度和高度监督信号。
2. 在模型中增加门热力图和门宽回归头。
3. 在 A、B 候选门之间做匹配。
4. 将匹配结果转换成当前工具使用的墙编号和墙内比例。
