失败样例：两个 head 都不可靠

这类样例是审稿人最容易质疑的地方：不是边界轻微偏移，而是布局拓扑或房间范围判断错误。平均指标不错，但长尾失败仍然明显。

审稿人视角的不足

只报 oracle 会被质疑，因为 oracle 是按 GT 事后选择 head，不能作为真实推理结果。
双头没有显式选择机制。test 中 original/new 谁更好接近五五开，说明需要 confidence 或 selector，而不是固定取某个 head。
损失主要是 depth、ratio、grad，对拓扑错误的惩罚不够；低 IoU 样例说明需要边界、角点、polygon consistency 或 IoU 类约束。
模型参数约 102M，训练集相对不大，且 dropout=0、weight_decay=0、无学习率衰减，容易出现 val 到 test 的泛化落差。

建议的下一轮实验

加一个 head selector/confidence head：训练它判断 original/new 哪个更可信，目标是逼近 oracle 的 0.8101。
启用更强泛化设置：AdamW + weight_decay 1e-4，dropout 0.1，cosine 或 plateau LR decay，早停在 380 到 440 epoch 附近。
增加几何一致性约束：低权重加入 BoundaryLoss、corner heatmap 或 polygon/IoU consistency，重点减少 IoU<0.5 的长尾失败。
做分层表格：按 room complexity、wall number、visible/full、occlusion 强度分组，证明改进不是只提高平均值。
