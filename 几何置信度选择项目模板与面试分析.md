# 基于几何一致性与置信度估计的全景 3D 房间布局估计改进

## 0. 当前结论

这个项目的核心不再是“再做一个更大的 layout 网络”，而是把 Bi-Layout 已有的 enclosed / extended 双分支输出变成一个更可靠、可部署的最终布局结果。

一句话版本：

```text
在 Bi-Layout 双分支预测基础上，引入 Geometry-aware Confidence Selector，根据预测布局的几何合法性、分支一致性和角点/边界置信度自动选择 enclosed 或 extended 分支，并对低置信度几何异常结果进行轻量修正，从而降低分支误选和无效布局输出。
```

## 2. 项目模板

### 项目名称

基于几何一致性与置信度估计的全景 3D 房间布局估计改进

### 项目背景

全景房间 Layout 估计需要从单张 360 度图像中恢复墙面边界、角点和三维结构。Bi-Layout 通过 enclosed / extended 双分支缓解了布局标注歧义，但实际预测仍会出现分支选择错误、角点跳变、边界不连续、自交和 extended 过度扩张等问题，影响下游室内建模、AR 测量和机器人空间理解。

### 问题定义

给定 Bi-Layout 的两个分支预测：

```text
P_origin = {depth, ratio}
P_new    = {new_depth, ratio}
```

设计一个轻量级 selector：

```text
selected_branch = Selector(geometry_features(P_origin, P_new))
```

并在低置信度或几何异常时执行修正：

```text
final_layout = RepairOrFallback(selected_branch, alternative_branch)
```

### 核心贡献

1. 构造 oracle selector label：根据 per-sample `full_3d` 或 `full_2d` 表现自动标注哪个分支更可信。
2. 设计 Geometry-aware Confidence Selector：输入仅使用预测布局本身的几何特征，不使用 GT 泄漏信息。
3. 加入几何一致性后处理：过滤自交、多余角点、边界突变和过度扩张。
4. 用 full_3d、3D IoU、selector accuracy、low-IoU/invalid rate 和失败案例可视化验证工程可靠性。

## 3. Geometry-aware Confidence Selector 设计

### 3.1 几何特征

建议第一版不用改 Bi-Layout 主干，只在推理输出上提取特征：

| 特征组 | 具体特征 | 作用 |
|---|---|---|
| Polygon 合法性 | `is_valid`、自交、面积、周长、最小边长、角点数 | 识别无效布局 |
| 边界平滑度 | depth 一阶/二阶差分均值、最大跳变、异常尖峰数量 | 识别角点跳变、边界断裂 |
| 分支一致性 | origin/new 面积比、depth L1、两分支 floorplan IoU | 判断两个分支是否冲突 |
| 过度扩张 | extended 面积 / enclosed 面积、最大半径比 | 过滤 extended 过度扩张 |
| 高度一致性 | ratio 合法范围、height IoU proxy | 防止高度异常 |
| 角点置信度 | corner heatmap 峰值；若未启用，则用 depth 梯度峰值代理 | 估计角点可靠性 |

注意：

```text
corner_heat_map 在当前模型里是可选输出，默认配置未启用。
第一版可以先用 depth 梯度峰值作为角点置信度 proxy；
第二版再打开 corner_heat_map 训练显式角点置信度。
```

### 3.2 Selector 模型

第一版建议从轻量模型开始：

```text
输入: 30-80 维几何特征
模型: Logistic Regression / MLP(2 layers) / RandomForest
输出: p_select_new
决策:
  p_select_new >= threshold -> choose new/extended
  otherwise -> choose original/enclosed
```

推荐先做两个 baseline：

```text
Naive-1: always choose original
Naive-2: always choose new
Naive-3: choose branch with lower invalid score
Ours: Geometry-aware selector
Oracle: GT selector upper bound
```

### 3.3 后处理策略

```text
if selected layout is invalid:
    if alternative layout is valid:
        fallback to alternative branch
    else:
        apply Manhattan post-processing

if extended area / enclosed area > threshold and confidence is low:
    prefer enclosed or shrink with boundary smoothing

if boundary jump count > threshold:
    smooth depth sequence or re-fit polygon

if polygon has self-intersection:
    use shapely buffer(0) repair or fallback to valid branch
```

## 6. 大厂面试官视角评价

### 6.1 项目亮点

1. 问题切得准：不是泛泛地“提高精度”，而是抓住 Bi-Layout 的真实短板：双头互补但缺少可部署 selector。
2. 工程价值明确：减少 invalid layout 和长尾失败，对 AR 测量、室内建模、机器人建图都能讲清楚价值。
3. 方法轻量：不重训大模型主干，先用几何特征和 selector 做稳定性提升，符合工业界“低成本提升可靠性”的思路。
4. 指标设计完整：不仅有 full_3d，还能讲 selector accuracy、invalid rate、low-IoU rate、失败案例分析。
5. 有上界论证：oracle full_3d=0.8101 证明 selector 问题有优化空间，不是拍脑袋加模块。

### 6.2 当前短板

1. 第一版 selector 已经训练并跑通，但目前是 test split 内 5-fold CV；更严格的版本应导出 train/val/test 三份 prediction，再做最终 test-only 评估。
2. invalid layout rate 还没有严格定义，目前主要用 low-IoU rate 和 polygon validity proxy 表示异常输出。
3. 角点置信度分支默认没启用，如果面试时强调 corner confidence，需要解释第一版用 depth 梯度 proxy，第二版启用 heatmap 或显式 corner score。
4. 几何 repair/fallback 还没有正式纳入最终指标，因此 full_3d low-IoU rate 暂时没有下降。
5. 只在 MatterportLayout test 上有数据，最好再补 ZInD 或复杂房间子集验证泛化。

### 6.3 面试官可能追问

```text
Q1: 你的 selector label 怎么构造？是否用了 GT？
答法: 训练阶段用 GT 指标构造 oracle label，推理阶段只用预测几何特征，不用 GT。

Q2: 你优化的是 full_2d 还是 full_3d？
答法: 正式版本同时构造 full_3d label 和 full_2d label。主结果看 full_3d，辅助报告 full_2d 和 low-IoU rate。

Q3: 为什么不用再训练一个端到端网络？
答法: 主干已经有较好预测，失败集中在分支选择和几何异常；轻量 selector 更可解释、成本低、风险小，也更适合工程部署。

Q4: 几何后处理会不会破坏正确样本？
答法: 只在低置信度或非法 polygon 上触发，并做 fallback；通过 invalid rate 和高 IoU 样本保持率做保护性评估。

Q5: 你如何证明不是过拟合测试集？
答法: selector 在 train/val prediction export 上训练和调参，test 只最终评估；按 scene split 做分组，补 ZInD 或 Atlanta subset 泛化验证。
```
