# 全景图布局识别系统（Bi-Layout）完整流程分析

## 系统概述
这是一个基于双头（Bi-Head）Transformer架构的全景室内布局识别系统，能够从单张全景图像预测室内房间的3D布局。

---

## 一、完整处理流程（从输入到输出）

### 1.1 训练/评估/推理流程总览

```
全景图像 (512x1024, RGB)
    ↓
[数据加载与预处理]
    ↓
[特征提取]（Backbone: ResNet50）
    ↓
[位置编码]（Position Embedding Sine）
    ↓
[双分支Transformer处理]
    ├─ 原始分支（Origin Head）
    └─ 新分支（New Head）
    ↓
[布局预测输出]
    ├─ depth (256维)
    ├─ ratio (1维 - 房间高度)
    └─ new_depth (256维，仅双头模式)
    ↓
[后处理（可选）]
    ├─ Manhattan Layout Fitting
    └─ 3D坐标转换
    ↓
最终输出：布局JSON / 3D模型
```

## 二、各主要模块详细说明

### 2.1 数据模块（Dataset）

#### 2.1.1 BaseDataset（基类）
**文件**: [dataset/communal/base_dataset.py](dataset/communal/base_dataset.py)

**职责**:
- 统一的数据处理接口
- 数据增强（旋转、翻转、拉伸）
- 标签转换

**输入输出**:
```
输入:
  - label: {corners, ratio, id, ...}  # 角点（UV坐标）
  - image: (H, W, 3) 全景图
  - patch_num: 256（默认）

处理:
  1. corners → visible_corners（处理遮挡的角点）
  2. visible_corners → depth向量（256维）
  3. 角点(UV) → XYZ坐标 → 深度图
  4. 数据增强（仅训练模式）
  5. 热力图生成（可选）

输出:
  - 'image': (3, 512, 1024) - 转置后的图像
  - 'depth': (256,) - 深度值序列
  - 'ratio': float - 房间高度比例
  - 'new_depth': (256,) - 新标签深度（双头模式）
  - 'opening': (1024,) - 开口mask（墙体缝隙）
  - 'corner_heat_map': (256,) - 角点热力图
```

#### 2.1.2 MP3DDataset
**文件**: [dataset/mp3d_dataset.py](dataset/mp3d_dataset.py)

**数据源**: Matterport3D数据集

**支持三种类型**:
- `'origin'`: 仅原始标签
- `'new'`: 包含新标签（遮挡处理）
- `'occlusion'`: 双头模式（原始+遮挡处理）

**数据路径结构**:
```
mp3d/
├── image/           # 全景图像 (PNG)
├── label/           # 标签 (JSON) - 包含corners和ratio
└── all_mix_labels_in_uv_v2/  # 遮挡标签 (TXT)
```

**关键操作**:
```python
# 示例：读取遮挡模式数据
getitem() → [img_path, label_path, new_label_path]
  → read_image() → (512, 1024, 3)
  → read_label() → {'corners': [[u,v], ...], 'ratio': float}
  → read_new_label() → 新的遮挡处理后的corners
  → process_data() → 标准化输出字典
```

#### 2.1.3 ZindDataset
**文件**: [dataset/zind_dataset.py](dataset/zind_dataset.py)

**数据源**: ZInD (Zillow Indoor) 数据集

**特殊处理**:
- VP对齐（Vanishing Point Alignment）- 自动旋转图像对齐视点
- 数据过滤：
  - `filter_center`: 验证相机在布局内
  - `filter_boundary`: 验证边界有效性
  - `filter_self_intersection`: 排除自相交的多边形

**配置选项**:
```python
is_simple=True          # 仅简单形状
is_ceiling_flat=False   # 平顶房间
vp_align=True           # 启用VP对齐旋转
```

### 2.2 模型模块（Models）

#### 2.2.1 Bi_Layout 主模型
**文件**: [models/bi_layout.py](models/bi_layout.py)

**架构流程**:

```
输入: (B, 3, 512, 1024)
  ↓
[特征提取器 - HorizonNetFeatureExtractor]
  输出: (B, 1024, 256)  # 1024维特征，256个patch
  ↓
[特征转置]
  输出: (B, 256, 1024)  # 便于Transformer处理
  ↓
[位置编码 - PositionEmbeddingSine]
  输出: (B, 256, 1024)  # 与特征相同维度
  ↓
[双分支Transformer处理]
  ├─ New Head Transformer
  │  输入: 特征(B,256,1024) + New查询嵌入(256,512)
  │  输出: (B, 256, 1024)
  │
  └─ Origin Head Transformer
     输入: 特征(B,256,1024) + Origin查询嵌入(256,512)
     输出: (B, 256, 1024)
  ↓
[双头输出处理]
  Origin分支:
    depth = Linear(1024→1) → (B, 256)
    ratio = Linear(1024→1) → Linear(256→1) → (B, 1)
  
  New分支:
    new_depth = Linear(1024→1) → (B, 256)
    new_ratio = Linear(1024→1) → Linear(256→1) → (B, 1)
  
  最终ratio = (ratio + new_ratio) / 2
  ↓
输出: {
  'depth': (B, 256),      # 原始布局深度
  'ratio': (B, 1),        # 房间高度比例
  'new_depth': (B, 256)   # 新布局深度（双头）
}
```

**关键参数**:
```python
backbone='resnet50'                 # 特征提取器骨干网络
patch_num=256                       # patch数量（分辨率）
feature_channel=1024                # 特征维度
embedding_channel=512               # 查询嵌入维度
height_compression_scale=8          # 高度压缩比例
depth=6                             # Transformer深度（层数）
win_size=8                          # 滑动窗口大小
output_number=2                     # 输出头数（1或2）
corner_heat_map=False               # 是否输出角点热力图
```

**核心Transformer解码器**:
- `Share_Feature_Guidance_Module`: 默认解码器
- 支持相对位置编码（RPE）和绝对位置编码（APE）

#### 2.2.2 特征提取器
**文件**: [models/modules/horizon_net_feature_extractor.py](models/modules/horizon_net_feature_extractor.py)

**功能**: 使用ResNet50提取全景图特征

**处理流程**:
```
输入: (B, 3, 512, 1024)
  ↓
ResNet50主干
  - 多层卷积和池化
  - 保留高分辨率特征
  ↓
高度压缩 (height_compression_scale=8)
  - 512 → 512/8 = 64 (高度)
  - 保留宽度 1024
  ↓
特征投影到统一维度
  ↓
输出: (B, feature_channel=1024, width=256)
```

### 2.3 损失函数模块（Loss）

**文件**: [loss/__init__.py](loss/__init__.py), [models/other/criterion.py](models/other/criterion.py)

**支持的损失函数**:

| 损失函数 | 说明 | 应用 |
|--------|------|------|
| `L1Loss` | 平均绝对误差 | depth预测 |
| `LEDLoss` | 边界梯度损失 | 布局边界平滑性 |
| `GradLoss` | 深度梯度损失 | 深度图连续性 |
| `BoundaryLoss` | 边界损失 | 角点位置准确性 |
| `BCEWithLogitsLoss` | 二元交叉熵 | opening预测（是否有缝隙） |
| `HeatmapLoss` | 热力图损失 | 角点位置热力图 |

**损失计算流程**:
```
gt (地真标签) + dt (模型预测)
  ↓
遍历所有配置的损失函数
  ├─ 计算单个损失: loss_i = criterion(gt, dt)
  ├─ 加权组合: weighted_loss_i = loss_i * weight_i
  └─ 累加: total_loss = Σ weighted_loss_i
  ↓
输出: {
  'loss': 总损失（用于反向传播）,
  'batch_loss_d': {各损失分量},
  'epoch_loss_d': 累计损失
}
```

**配置示例** (YAML):
```yaml
CRITERION:
  depth:
    NAME: depth
    LOSS: L1Loss
    WEIGHT: 1.0
    WEIGHTS: [1.0]
  
  grad:
    NAME: grad
    LOSS: GradLoss
    WEIGHT: 0.5
    WEIGHTS: [0.5, 0.5, 0.5]
  
  opening:
    NAME: opening
    LOSS: BCEWithLogitsLoss
    WEIGHT: 0.3
    WEIGHTS: []
```

### 2.4 后处理模块（Post-processing）

**文件**: [postprocessing/post_process.py](postprocessing/post_process.py)

**功能**: 将原始深度预测转换为规则化的布局

**支持三种方法**:

```
原始深度值 (256维)
  ↓
转换为XYZ坐标
  ├─ depth2xyz(): 深度 → 3D坐标
  └─ 输出: (256, 3)
  ↓
平面化处理
  ├─ xyz[..., ::2]: 提取X-Z平面（忽略Y高度）
  └─ 输出: (256, 2) - 2D楼层平面
  ↓
规则化方法选择:
  
  1. Manhattan (默认)
     fit_layout(): 拟合到曼哈顿世界假设
     ↓ 输出: 矩形或L形规则化布局
  
  2. Manhattan (旧版本)
     fit_layout_old(): 传统Manhattan算法
  
  3. Atalanta
     cv2.approxPolyDP(): 多边形近似
  ↓
恢复3D坐标
  ├─ 在Y=1（地面）处插入高度
  └─ 输出: (N_corners, 3) - 最终3D角点
  ↓
输出: processed_xyz (可变数量角点)
```

**参数**:
- `need_cube=False`: 是否强制立方体形状（仅4个角点）

## 三、数据流向详细追踪

### 3.1 训练流程数据流
```
Step 1: 数据加载
  数据集 → 批处理 (batch_size)
  输出: batch = {
    'image': (B, 3, 512, 1024),
    'depth': (B, 256),
    'ratio': (B, 1),
    'new_depth': (B, 256),  # occlusion模式
    'opening': (B, 1024),   # occlusion模式
    'corner_heat_map': (B, 256),  # 可选
    'id': List[str]
  }

Step 2: 数据转移到设备
  batch → CUDA (if available)
  数据类型转换 (float32 → float16 if AMP)

Step 3: 前向传播
  model(images) → outputs = {
    'depth': (B, 256),
    'ratio': (B, 1),
    'new_depth': (B, 256)
  }

Step 4: 损失计算
  calc_criterion(criterion, gt, dt) →
  {
    'loss': scalar,  # 总损失
    'batch_loss_d': {
      'depth': float,
      'ratio': float,
      'grad': float,
      'opening': float,
      ...
    }
  }

Step 5: 反向传播和优化
  loss.backward()
  optimizer.step()
  lr_scheduler.step() (per epoch)

Step 6: 验证（每个epoch）
  无梯度评估
  计算IoU指标 (2D/3D, visible/full)
  计算高度误差
  保存最佳模型
```

### 3.2 验证/测试流程数据流
```
Step 1: 加载测试数据
  dataset['image', 'depth', 'ratio', 'new_depth', 'opening', ...]

Step 2: 前向传播（无梯度）
  model(images) → dt = {
    'depth': (B, 256),
    'ratio': (B, 1),
    'new_depth': (B, 256)
  }

Step 3: 后处理
  dt['depth'] → depth2xyz() → processed_xyz (3D坐标)
  post_process() → 规则化布局
  
Step 4: 评估
  计算多维度IoU:
    - visible_2d/3d: 可见部分
    - full_2d/3d: 完整布局
    - height: 高度误差
  
  计算其他指标:
    - CE (Corner Error): 角点位置误差
    - PE (Pixel Error): 像素误差
    - F1-Score: 角点检测F1
    - RMSE: 均方根误差

Step 5: 结果汇总和保存
  results = {
    'visible_3d': 0.85,
    'full_3d': 0.82,
    'height': 0.15,
    ...
  }
  保存 → JSON文件
```

### 3.3 推理流程数据流
```
Step 1: 加载单张图像
  img_path → Image.open() → (H, W, 3)
  resize → (512, 1024, 3)
  
  可选: VP对齐
    panoEdgeDetection() → 消失点检测
    rotatePanorama() → 图像对齐
    归一化: img / 255.0

Step 2: 模型推理
  img (512, 1024, 3) 
    → 增加batch维: (1, 3, 512, 1024)
    → model()
    → dt = {
        'depth': (1, 256),
        'ratio': (1, 1)
      }

Step 3: 坐标转换
  depth (1, 256) → depth2xyz() → xyz (256, 3)
  ratio (1, 1) → 房间高度 (标量)

Step 4: 后处理（可选）
  xyz → fit_layout() → processed_xyz
  保存处理前后的布局

Step 5: 输出生成
  1. 可视化:
     - 2D边界线绘制
     - 楼层平面图
     - 梯度/深度可视化
     → 保存PNG
  
  2. 3D输出:
     - xyz → corners2boundaries() → 边界
     - layout2depth() → 深度图
     - create_3d_obj() → OBJ模型
     → 保存OBJ/PLY
  
  3. JSON输出:
     - xyz2json() → JSON格式
     → 保存JSON
```

## 五、数据坐标系统说明

### 5.1 UV坐标系（图像空间）
```
原点 (0, 0)  --->  右  --->  (1, 1)
                               |
                               v
                             下
```
- u ∈ [0, 1]: 水平（图像宽度）
- v ∈ [0, 1]: 竖直（图像高度）
- 常用于标注和热力图

### 5.2 XYZ坐标系（3D世界空间）
```
Y (竖直 - 高度)
^
|    /Z (深度 - 前后)
|  /
|/-----> X (水平 - 左右)
```
- 相机在原点
- 地面在Y=0
- 天花板在Y=1（标准化）
- X-Z平面投影用于楼层平面图

### 5.3 Depth向量
```
depth: [d_0, d_1, ..., d_255]
  |
  └─ 256个离散样本点
  └─ 均匀分布在水平方向（u）
  └─ 每个值对应该方向的深度（0-1标准化）
```

### 5.4 坐标转换关系
```
UV坐标 ← uv2xyz() → XYZ坐标
        ← depth2xyz() ←
        
UV坐标 ← uv2pixel() → 像素坐标 (0~1024)
```

## 八、关键概念与术语

| 术语 | 定义 |
|------|------|
| **Panoramic Image** | 360°全景图，通常分辨率 512×1024 |
| **Layout** | 房间的3D几何结构（角点+高度） |
| **Depth** | 深度值序列，256个样本点 |
| **Ratio** | 房间高度与相机高度的比例 |
| **Bi-Head** | 双输出头（原始布局+遮挡处理） |
| **Occlusion** | 物体遮挡导致的不完整布局标签 |
| **VP Alignment** | 消失点对齐，规范化图像方向 |
| **Manhattan Layout** | 所有墙面垂直或平行的房间假设 |
| **IoU** | 交并比，布局预测准确率指标 |
| **Visible Corners** | 从单个视点可见的角点 |
| **Full Corners** | 完整的房间角点（包括不可见部分） |
